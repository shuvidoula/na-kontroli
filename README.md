# НА-КОНТРОЛІ

**НА-КОНТРОЛІ** - мобільний PWA-задачник для швидкого особистого контролю задач, доручень і виконання. Додаток зроблений як простий інструмент: відкрив, додав задачу, поставив час, розбив на етапи, передав іншому виконавцю або закрив як виконану.

Поточний реліз: **1.0.6**.

Працює через браузер, встановлюється на екран телефону як додаток і після першого кешування може відкриватися офлайн. Дані задач зберігаються локально на пристрої, а не на сервері.

## Для чого

- швидко записати задачу на сьогодні або на конкретний час;
- бачити окремо **Мої**, **Іншим**, **Всі** та **Готово**;
- додавати теги **Особисте**, **На контролі**, **Терміново** і швидко фільтрувати список;
- створювати швидку задачу одним рядком на кшталт `14:30 перевірити звʼязок. коротка примітка`;
- розкладати задачу на етапи виконання;
- контролювати доручені задачі;
- фіксувати, кому передано задачу;
- отримувати локальні нагадування за 1 годину до часу задачі та в сам час задачі;
- мати резервну копію через експорт/імпорт.

## Головні можливості

**Локальні сховища.** На одному телефоні можна створити кілька окремих сховищ. Кожне відкривається своїм 4-значним PIN.

**Мобільний інтерфейс.** Основні кнопки винесені вниз: `Мої`, `Іншим`, `+`, `Всі`, `Довідник`. Додаткові дії знаходяться у верхньому меню.

**Швидка і повна задача.** Кнопка `+` дає вибір: швидка задача для одного рядка з датою і тегом або повна задача з усіма полями, етапами та дорученням.

**Етапи задачі.** У задачу можна додавати етапи, відмічати їх виконання і змінювати порядок у режимі правки.

**Теги контролю.** Для задач можна вибрати тег: `О` - особисте, `К` - на контролі, `Т` - терміново.

**Прострочення.** Якщо дата і час минули, а задача не виконана, вона підсвічується червоним.

**Доручення іншим.** Якщо в задачі заповнено поле виконавця, вона переходить у напрямок `Іншим`. Так простіше відділити особисті задачі від переданих на контроль.

**Передача задачі.** Кнопка `Передати` відкриває поле `Кому передано`. Після підтвердження задача переходить у `Іншим`, отримує мітку передачі та виконаний етап `Передано: ...`.

**Офлайн-режим.** Файли додатку кешуються service worker. Після встановлення додаток може запускатися без постійного інтернету.

## Приватність

- Задачі зберігаються в `localStorage` браузера на конкретному пристрої.
- GitHub Pages віддає тільки файли додатку.
- Додаток не має серверної бази даних.
- У `Content-Security-Policy` вимкнені зовнішні підключення через `connect-src 'none'`.
- Бібліотеки лежать локально в репозиторії, без CDN.

Важливо: це не військова сертифікована система захисту інформації. Не передавайте через додаток секретні, критичні або такі дані, які за правилами мають іти тільки через офіційні захищені канали.

## Як встановити

1. Відкрийте GitHub Pages URL цього репозиторію у Safari на iPhone або Chrome на Android.
2. На iPhone: `Поділитися` -> `На екран Домівки`.
3. На Android: меню Chrome -> `Встановити додаток` або `Додати на головний екран`.
4. Створіть назву сховища і PIN.

## Швидке створення задачі

1. Натисніть `+`.
2. Оберіть `ШВИДКА`.
3. Дата одразу стоїть сьогодні. Її можна змінити в полі дати.
4. Впишіть задачу одним рядком.
5. За потреби виберіть тег: `О`, `К`, `Т` або `Без тегу`.

Підтримані формати часу:

- `14:30 перевірити звʼязок. коротка примітка`
- `14 перевірити звʼязок. коротка примітка`
- `14 30 перевірити звʼязок. коротка примітка`

Текст до першої крапки стає назвою задачі. Текст після крапки стає приміткою. Після створення швидку задачу можна відкрити через `Правка` і редагувати як повну.

## Бекап та імпорт

- `Бекап` створює JSON-файл або відкриває системне меню поширення на телефоні.
- Якщо телефон не зберіг файл, додаток показує текст резервної копії і копіює його в буфер.
- `Імпорт -> Додати` залишає поточні задачі та додає задачі з файлу.
- `Імпорт -> Замінити` видаляє поточні задачі та ставить задачі з файлу.

## Як працює передача

1. Відкрийте задачу і натисніть `Передати`.
2. Вкажіть імʼя, позивний або підрозділ у полі `Кому передано`.
3. Натисніть `Завершити передачу`.
4. Задача перейде у вкладку `Іншим`, а всередині задачі зʼявиться виконаний етап `Передано: ...`.
5. Після реального виконання відкрийте задачу і натисніть `Зроблено`.

## Технічна основа

- Framework7 Core `9.x`
- Static GitHub Pages
- `localStorage` для локальних сховищ і задач
- `service worker` для офлайн-кешу
- локальні бібліотеки в `vendor/`, без CDN

## Структура

- `index.html` - HTML-оболонка PWA
- `src/css/app.css` - стиль додатку
- `src/js/crypto.js` - локальне кодування, hash, утиліти
- `src/js/storage.js` - сховища, PIN, задачі, сумісність зі старими даними
- `src/js/app.js` - екрани, задачі, передача, імпорт/експорт
- `vendor/framework7/` - локальний Framework7 bundle
- `manifest.webmanifest` - опис для встановлення
- `sw.js` - офлайн-кеш і оновлення

## Розгортання на GitHub Pages

1. Завантажте файли цієї папки в корінь репозиторію.
2. Не завантажуйте `node_modules/`; потрібний Framework7 вже лежить у `vendor/framework7/`.
3. Відкрийте `Settings` -> `Pages`.
4. Source: `Deploy from a branch`.
5. Branch: `main`, folder: `/root`.
6. Після деплою відкрийте URL виду `https://username.github.io/repo-name/`.

## Оновлення PWA

Після зміни коду піднімайте версію кешу в `sw.js`:

```js
const CACHE_NAME = "na-kontroli-pages-v1.0.6";
```

Так встановлена PWA на телефоні підтягне нові файли. Дані задач залишаються в локальному сховищі браузера.
